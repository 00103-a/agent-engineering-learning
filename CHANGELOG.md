# 学习设计变更记录

用于记录学习路线、能力边界和设计文档的重要变化。普通错别字和格式调整不需要记录。

## 2026-07-14

### 建立学习知识库

- 确定 2027 年寒暑假实习为阶段目标。
- 确定 Agent 应用工程为当前主线，Python 后端作为主要承载，全栈能力后续逐渐增加。
- 确定练习代码独立，不读取或修改 DayAgent。
- 建立 Agent 六层能力结构和九类能力优先级。
- 建立未来可迁移的工程原则，避免围绕框架 API 学习。
- 建立交互式源码学习方式，要求预测、实现、测试、对比和复述。
- 增加设计文档每日按需更新和阶段复盘机制。

### 当前状态

- 学习边界已经形成初稿。
- 尚未正式开始第一阶段课程。

### 添加课前诊断

- 新增 Agent 工程基础短答与异步执行推演。
- 新增独立的最小 Agent Runtime 编码骨架。
- 新增工具注册、超时、错误分类、重试、终止和步骤上限测试。
- 诊断结果将用于确定第一阶段课程起点，不作为单纯的通过/失败考试。

## 2026-07-15

### 根据课前诊断调整第一阶段路线

- 记录诊断的 7 个测试通过，但实现依赖逐步提示，暂不认定为独立掌握 Runtime。
- 将当前主线明确为“Agent 工程，当前训练重点为 Python-for-Agent”。
- 增加字典、数据类、异常、异步和 pytest 的 Agent 场景化训练。
- Runtime 改为“看懂修改、根据测试补全、独立重写”三级递进。
- 框架源码、上下文、记忆和 MCP 暂缓到阶段验收之后。
- 新增 2026-07-15 至 2026-07-28 两周计划，每天安排 1～2 小时核心深度练习。
- 取消对 LangGraph 的永久主框架绑定，改为进入框架阶段时按维护状态和岗位需求重新选择。
- 新增 2026-07-15 框架状态记录，保留 LangGraph 与 OpenAI Agents SDK 的官方核实来源。

## 2026-07-18

### 调整独立编码的复现方式

- 学习者反馈立即从空白重写同一道综合题会打击学习积极性。
- 将日常训练改为螺旋式复现：新任务约 70% 学习新目标，约 30% 嵌入旧能力。
- 优先在不同工具和 Runtime 场景中检验近迁移，不把重复原题作为日常验收。
- 完整空白重写保留为后续阶段验收，在建立连续成功体验后安排。

## 2026-07-21

### 完成异步执行实验并增加双轨学习结构

- 新增串行与并发查询、超时转换、取消清理、完成顺序消费和并发上限实验。
- 新增并发查询与安全超时结果测试。
- 每日学习调整为编码主练习与 Agent 知识输入双轨；纯听讲内容只记为“已接触”。
- Agent 知识输入覆盖多工具 Runtime、Agent 与 Workflow 边界以及上下文工程。
- 新增可持续更新的下一课学习上下文，明确当前能力证据、未验收知识和下一次课程的开场、产出与停止条件。

## 2026-07-22

### 将短期安排升级为滚动总体计划

- 根据 5 个正式学习日的实际进度，取消按原两周日历强制推进，改为按有效学习日和能力证据验收。
- 当前投入调整为 Agent 核心编码 65%、Python 与测试复现 20%、全栈知识 15%。
- 明确八月底以 Agent 基础达到可迁移为主，全栈仅轻量穿插，不要求独立掌握。
- 增加“已接触、跟随完成、可迁移、可交付”四级证据，避免把 AI 生成或 C/D 级带写代码误判为掌握。
- 总体路线改为 Agent 与全栈轻量交织，Agent 核心验收后再提高全栈产品化比例。
