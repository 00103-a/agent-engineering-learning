# 下一课学习上下文

> 更新时间：2026-07-22。此文件是下一次教学的首要入口；开始课程前先读取本文件、`00-学习设计文档.md`、`01-两周学习计划.md` 和最近一篇学习记录。

## 1. 学习目标与边界

- 求职方向：Agent 工程师或全栈工程师，重点准备 2027 年寒暑假实习。
- 当前主线：Agent 工程，使用 Python 工程能力承载 Runtime、工具执行、上下文和记忆等主题。
- 每天可投入 3～4 小时，其中适合深度编码的时间为 1～2 小时。
- 当前不读取或改造 DayAgent 代码，也不默认讨论对 DayAgent 的启发。
- 不追逐单一框架 API；优先掌握工具契约、执行状态、超时、并发、错误、预算和测试等稳定抽象。
- 当前阶段按 Agent 核心编码 65%、Python 与测试复现 20%、全栈知识 15% 分配；全栈不作为八月底独立编码验收项。
- 截至 2026-07-22 共完成 6 个正式学习日；计划按有效学习日推进，不按日历虚增进度。

## 2. 教学方式

- 每日双轨：45～70 分钟编码主练习；精力有余时增加 20～40 分钟 Agent 知识输入。
- 每次编码只引入一个主要新机制，新任务约 70% 新内容、30% 复用旧能力。
- 不立即重写刚完成的原题。通过不同场景中的近迁移复现旧机制，完整重写留作后续阶段验收。
- 学习者先写；卡住后依次提供概念提示、结构提示、局部代码，最后才给完整示例。
- 纯听讲内容只标记为“已接触”，不能算作会实现或独立掌握。
- 以实际完成状态推进，不根据日期或 `Day N` 自动跳课。
- 允许 AI 主写样板和普通全栈代码；Agent 关键路径只有在学习者能够预测、修改、测试和解释后才提升掌握等级。

## 4. 已接触但未验收的 Agent 知识

- 多工具 Runtime：模型提出调用，Runtime 负责验证、依赖、并发、超时、重试、身份和结果记录。
- Agent 与 Workflow：路径固定时由程序编排；路径需根据 Observation 动态变化时才让 Agent 决策。
- 上下文工程：记忆负责保存信息，上下文工程负责为当前模型调用选择信息。
- 工具依赖图与失败策略：理解 `failed`、`blocked`、fail-fast、best-effort 和 fallback 的用途。
- Trace 与评估：理解详细轨迹主要服务调试和评估，Observation 只保留模型下一步所需信息。

以上内容下一课不得直接当作学习者已经会设计或实现。

## 5. 下一课核心主题

**带身份、错误分类和尝试次数的有限重试。**

选择这个主题的原因：有限重试曾在逐步提示下完成，但尚未在新场景中独立迁移。下一课只新增“尝试状态如何进入结构化结果”，继续复用已经理解的 `call_id` 和异常边界。

### 新旧比例

- 新内容约 70%：错误类型、`attempts`、可重试判断和重试耗尽语义。
- 旧内容约 30%：异步 Handler、`call_id`、结构化结果、异常捕获和 pytest。
- 不加入并发批次、Registry 或 Agent Loop，避免重试策略与调度策略同时出现。
- 全栈只在精力有余时继续请求模型与参数验证知识，不设编码要求。

## 6. 下一课编码产出

新建独立场景，不修改前两次异步练习：

```text
exercises/2026-07-23-retry-policy/
├── retry_policy.py
└── test_retry_policy.py
```

目标闭环：

1. 暂时性错误可以在预算内重试，永久性错误立即返回。
2. `max_retries` 与总尝试次数的关系由测试固定。
3. 最终结果保留 `call_id`、`attempts`、`status`、`error_type`、`value` 和 `error`。
4. 覆盖首次成功、暂时失败后成功、重试耗尽和永久失败。
5. 学习者至少独立完成一次循环边界、一个错误分支和两个断言。

不把完整 Registry、并发批次和 Agent Loop 一次性塞入本课。

## 7. 下一课开场

先给出以下场景，不直接给代码：

```text
call-search 第一次抛出 TransientToolError
call-search 第二次返回搜索结果
max_retries = 1
```

开场问题：最终结构化结果中的 `attempts` 应该是多少，为什么？

随后提供固定的异常类和结果数据类，由学习者一次只完成成功路径、暂时错误、永久错误中的一个分支。

## 8. 下一课验收与停止条件

- 能解释 `max_retries=1` 为什么最多尝试两次。
- 能让永久错误不重试，让暂时错误只在预算内重试。
- 能通过 `call_id` 和 `attempts` 说明最终结果属于谁、执行过几次。
- 若循环与异常缩进连续两次需要 C/D 级提示，先退回同步函数，不叠加超时。
- 编码完成后可只听一个 Agent 主题：重试、重规划和 fallback 的职责边界；不要求当日实现。
